# Bug Reports Portfolio – PrestaShop

This repository showcases a curated collection of **20 professionally written bug reports** based on the PrestaShop demo application. The purpose of this repo is to demonstrate practical QA skills, real-world defect reporting, structured investigation, and clear communication—key qualities expected from QA Engineers and SDETs.

These reports are written in a way that:

* Reflects real-world QA practices

* Shows understanding of severity, priority, and business impact

## ** What These Bug Reports Cover**

The defects span across multiple areas of PrestaShop:

### **Functional Issues**

* Add to cart inconsistencies
* Incorrect sorting
* Checkout validation gaps

### **UI/UX Issues**

* Broken layouts
* Misaligned elements
* Missing tooltips or feedback

### **Performance Issues**

* Slow page loads
* Delays in updates

### **Validation & Data Issues**

* Inputs accepting invalid values
* Missing required field checks

### **Navigation & Flow Issues**

* Incorrect redirects
* Breadcrumb inconsistencies

These issues simulate the type of bugs testers are expected to find in real e-commerce applications.

## **Bug Report Format Used**

Every bug in this portfolio includes:

* Title
* Module / Area
* Severity
* Environment
* Clear steps to reproduce
* Expected vs actual results
* Notes / Attachments

This aligns with standard bug-tracking tools like:

* JIRA
* Azure DevOps
* Bugzilla
* YouTrack

---

## **Severity Guide**

A simple severity scale used throughout the documentation:

| Severity     | Meaning                                    |
| ------------ | ------------------------------------------ |
| **Critical** | Breaks major functionality / no workaround |
| **High**     | Affects essential user actions             |
| **Medium**   | Impacts usability or non-core flows        |
| **Low**      | Minor UI/UX or non-blocking issues         |
